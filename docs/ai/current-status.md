Разработка начинается с **исследования источников**, а не с полной архитектуры.

# Sandbox

Для исследования используется отдельный проект:

Pika.Sandbox

Это временный Web API проект.

В нем будут находиться:

- экспериментальные модели
- черновая бизнес-логика
- тестовый парсинг
- HTTP запросы к источникам

Sandbox может содержать код, который **не соответствует финальной архитектуре**.

Его задача — исследование.

# Текущий Flow

Описан в данном [файле](flow.excalidraw)

Это excalodraw файл, в котором я на примитивных схемах попытлся описать то, как я вижу текущий поток данных в приложении

В нем я выделил 2 основных JSON файла. 

Request JSON:
```json
{
  "url": "ссылка на тайтл и серию"
}
```
Данный JSON отвечает за отправку через API ссылки, по которой будет происходить поиск данных

Response JSON

```json
{
  "sourceUrl": "...",
  "players": [
    {
      "name": "kodik",
      "data": [
        {
          "voice": "AniDUB",
          "quality": "720p",
          "m3u8Url": "...",
          "resolvedAt": "2026-03-11T10:00:00Z"
        }
      ]
    }
  ]
}
```

Это необходимый ответ от API. В нем содержатся вся необходимая информация такая как:
- изначальная ссылка на аниме
- список объектов плеера
  - наименование плеера
  - список данных плеера
    - озвучка
    - качество
    - ссылка потока
    - время получения ссылки (т.к. она временная)
